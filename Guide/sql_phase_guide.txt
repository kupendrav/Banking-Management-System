POSTGRESQL ENHANCEMENTS - PHASE WISE GUIDE
=======================================

PHASE 1: TRIGGERS
---------------
Triggers automatically run when certain events happen in the database.

1. Prevent Negative Balance Trigger
   - What it does: Stops accounts from having negative balances
   - When it runs: Before updating account balance
   - Why it's useful: Ensures data integrity and prevents overdrafts

2. Auto Timestamp Trigger
   - What it does: Automatically updates the "last modified" time
   - When it runs: Before updating any account information
   - Why it's useful: Keeps track of when accounts were last changed

3. Withdrawal Limit Trigger (Commented Out)
   - What it does: Blocks withdrawals over $10,000
   - When it runs: Before inserting into audit table
   - Why it's useful: Adds security for large transactions

PHASE 2: MATERIALIZED VIEWS
------------------------
Materialized views are like saved queries that update periodically.

1. Account Deposits View
   - Shows total deposits for each account
   - Updates when you refresh the view

2. Account Withdrawals View
   - Shows total withdrawals for each account
   - Helpful for account analysis

3. Recent Transactions View
   - Shows last 10 transactions
   - Good for quick monitoring

4. Most Active Users View
   - Shows accounts with most transactions
   - Useful for identifying frequent users

PHASE 3: INDEXES
--------------
Indexes make database queries faster.

1. Account Number Index
   - Speeds up searches by account number
   - Applied to both accounts and audit tables

2. Timestamp Index
   - Makes time-based searches faster
   - Good for transaction history queries

3. Action Index
   - Speeds up searches by transaction type
   - Helps with filtering deposits vs withdrawals

4. Composite Index
   - Combines account number and action
   - Optimizes common query patterns

PHASE 4: STORED PROCEDURES
-----------------------
Procedures are reusable sets of SQL commands.

1. Delete Old Audits Procedure
   - Removes audit records older than 1 year
   - Helps keep database size manageable

2. Monthly Report Procedure
   - Generates summary of monthly activity
   - Shows total deposits, withdrawals, and active accounts

PHASE 5: ADVANCED QUERIES
----------------------
Subqueries for data analysis.

1. High Value Accounts Query
   - Finds accounts with deposits over $10,000
   - Uses subqueries to aggregate data

2. Active Accounts Query
   - Lists accounts with more than 5 transactions
   - Helps identify engaged users

3. Most Active User Query
   - Shows the account with the most transactions
   - Single result showing top user

PHASE 6: TABLE RELATIONSHIPS
-------------------------
Foreign key relationships between tables.

1. Accounts → Audit Relationship
   - Each audit record links to one account
   - One-to-many: One account can have many audit records
   - Ensures data consistency

PHASE 7: MAINTENANCE COMMANDS
--------------------------
Commands for database upkeep.

1. Refresh Materialized Views
   - Updates all the saved views with current data
   - Should run regularly to keep stats current

2. Backup/Restore Commands
   - For protecting your data
   - Run manually in pgAdmin or command line

HOW TO USE THESE FEATURES
----------------------
1. Run the SQL file in your PostgreSQL database
2. The features will be automatically applied
3. Some features (like triggers) work automatically
4. Others (like views) need to be refreshed periodically
5. Procedures can be called when needed
